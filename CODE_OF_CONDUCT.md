# BetterNAT Code Of Conduct

## Purpose

BetterNAT is infrastructure software. Good technical discussion is direct,
evidence-based, and respectful of the people doing the work.

This code of conduct applies to project spaces including issues, pull requests,
discussions, documentation, and maintainer-operated community channels.

## Expected Behavior

- Be respectful and professional.
- Keep technical criticism specific and actionable.
- Assume good intent while still asking for evidence.
- Stay on topic for the issue, pull request, or discussion.
- Disclose operational risk clearly when discussing production-like changes.
- Respect security reporting boundaries described in `SECURITY.md`.

## Unacceptable Behavior

- Harassment, threats, or personal attacks.
- Discriminatory language or behavior.
- Publishing private information without explicit permission.
- Repeated off-topic comments after maintainers ask to refocus.
- Bypassing responsible disclosure for security-sensitive findings.
- Pressuring maintainers or users to run unsafe infrastructure commands.

## Enforcement

Maintainers may remove comments, close issues, restrict participation, or block
users when behavior harms the project or its contributors.

For conduct concerns, contact the maintainers using the reporting path in
`SECURITY.md` if the issue is sensitive. For non-sensitive issues, open a GitHub
issue with enough context for maintainers to review.

Maintainers should handle reports with discretion and should avoid public
disclosure of private reporter details.
