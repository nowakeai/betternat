# BetterNAT v0.2.0

Status: draft release notes. Do not publish until runtime, provider, AWS module,
GCP module, Registry, and disposable smoke validation are complete.

## What Changed

- Terraform surface reset to cloud-specific resources:
  `betternat_aws_gateway` and `betternat_gcp_gateway`.
- Module-first install direction:
  `nowakeai/betternat/aws` and `nowakeai/betternat/google`.
- GCP support added for Firestore-backed HA, tagged route ownership, LoxiLB
  datapath bootstrap, zonal MIG capacity repair, and optional stable public
  identity through an existing regional static external IPv4 address.
- GCP operations support added for `betternat status`, `doctor --live`,
  handover history, support bundle context, route checks, and Firestore-backed
  coordination.
- BetterNAT's product datapath contract clarified: LoxiLB is the supported
  datapath; nftables is not a product fallback path.

## Who Should Use It

Use this release for new BetterNAT Terraform installs after reading the cloud
quick start and limitations:

- AWS: `docs/user/getting-started/QUICK_START.md`
- GCP: `docs/user/getting-started/GCP_QUICK_START.md`
- Limits: `docs/user/reference/LIMITATIONS.md`

Do not treat this as an in-place upgrade from an older Terraform resource shape.
Plan replacements carefully and review route ownership before touching existing
egress paths.

## Validation Required Before Publication

- Runtime Go tests and builds.
- Provider Registry install validation.
- AWS module Registry page validation.
- GCP module Registry page validation.
- Disposable AWS module smoke with private-client egress, handover, destroy,
  and residual scan.
- Disposable GCP module smoke with Firestore HA, private-client egress,
  handover, stable public identity when configured, destroy, and residual scan.

## Known Limitations

- BetterNAT is self-managed and does not provide AWS NAT Gateway or Google Cloud
  NAT equivalent SLA semantics.
- Active flows may reset during failover.
- AWS gateway groups are one AZ per HA group.
- GCP gateway groups are one zone per HA group.
- GCP stable public identity is connectivity-first: successful new connections
  may temporarily use the target gateway's ordinary public IP before the
  regional static external IPv4 address converges back.
- Published BetterNAT AMIs are still outside this release scope.

## Artifact Integrity

Use checked-in release assets and SHA256 checksums from the matching GitHub
Release. Terraform Registry provider and module installs should use pinned
versions. Do not use pre-release artifact URL overrides in normal user installs.
