# BetterNAT v0.2.1

Draft status: do not publish until the final release-artifact validation in
`docs/release/GCP_GA_USER_DOCS_CHECKLIST.md` is complete.

## What Changed

- Hardened GCP runtime service account creation and reuse during gateway
  replacement.
- Added retry coverage for GCP IAM binding propagation errors where the runtime
  service account exists but IAM policy writes temporarily reject it as missing,
  deleted, or disabled.
- Retained provider-managed GCP runtime service accounts during gateway cleanup
  so same-name replacement does not hit GCP service-account delete/recreate
  propagation windows. BetterNAT-managed IAM bindings are still removed during
  cleanup.
- Improved agent startup state reporting so LoxiLB, route, and lease
  convergence during a fresh boot reports as `INIT` until the process has
  reached healthy `ACTIVE`. Post-ready failures still report `DEGRADED`.
- Updated GCP cleanup and IAM docs to explain retained runtime service accounts
  and manual removal after all gateways using the account are destroyed.

## Who Should Use It

Use this patch release after it is published if you run or evaluate BetterNAT on
GCP, especially with repeated gateway replacement, GKE private-node egress, or
small gateway machine types.

AWS users do not need this patch for an AWS-specific fix, but can adopt it as
the current v0.2 runtime once the release validation is complete.

## Known Limitations

- BetterNAT is self-managed and does not provide AWS NAT Gateway or Google Cloud
  NAT equivalent SLA semantics.
- Active flows may reset during failover.
- AWS gateway groups are one AZ per HA group.
- GCP gateway groups are one zone per HA group.
- Provider-managed GCP runtime service accounts are retained during gateway
  cleanup. Operators should remove them manually only after all gateways using
  the account are destroyed.
- GCP stable public identity is connectivity-first: successful new connections
  may temporarily use the target gateway's ordinary public IP before the
  regional static external IPv4 address converges back.
- Published BetterNAT AMIs are still outside this release scope.
