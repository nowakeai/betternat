# v0.1.0-alpha AWS Release Candidate Results

Date: 2026-06-21

## Summary

The `v0.1.0-alpha.test` build passed the alpha AWS acceptance scope in disposable `us-west-2a` fixtures.

The run used the current Terraform provider, current cloud-init bootstrap path, current `betternat-agent`, and LoxiLB/eBPF as the primary datapath. It intentionally did not run expensive multi-TB throughput tests.

Passed:

```sh
go test ./...
```

The Terraform validation commands required normal provider plugin socket access. They passed outside the restricted sandbox.

## Script And Environment Failures

Non-product failures encountered and fixed during the acceptance pass:

- zsh `status` is read-only, so the script variable was renamed,
- `betternat doctor` does not support `--format json`, so the script now uses the current CLI shape,
- one non-stable attempt hit an AWS STS `GetCallerIdentity` timeout before creating resources,
- ASG replacement parsing initially mishandled tab-separated AWS CLI output and was fixed.

All partial AWS resources from failed or interrupted runs were destroyed or confirmed absent.

## Not Covered

Not covered by this release-candidate pass:

- expensive multi-TB transfer benchmarks,
- production AMI boot-to-ready timing,
- active connection preservation,
- multi-AZ or multi-cloud behavior,
- LoxiLB eBPF map capacity tuning,
- ENA/RSS/IRQ advanced tuning.

## Conclusion

The alpha AWS acceptance criteria are satisfied for Terraform plus cloud-init deployment in disposable AWS environments.

The key remaining release work is packaging and documentation polish, not another alpha-blocking datapath or HA proof. Production readiness still requires AMI packaging, repeated benchmark-backed performance work, stronger upgrade/rollback validation, and broader operational hardening.
