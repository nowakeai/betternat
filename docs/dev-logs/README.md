# Dev Logs

Use this directory for durable implementation notes, session summaries, and architecture pivots.

Recommended structure:

- `YYYY-MM/` for monthly topic notes.
- `YYYY-MM-DD-short-topic.md` for one-off summaries.

Add a dev-log when a session produces reusable knowledge that is too detailed for `README.md` or `docs/architecture.md`, especially:

- cloud spike results,
- LoxiLB/nftables behavior,
- AWS cleanup lessons,
- dependency upgrade decisions,
- harness or workflow changes,
- non-obvious bugs and fixes.
